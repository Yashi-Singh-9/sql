# FROM

The `FROM` clause in SQL specifies the table or tables from which data should be retrieved. It is a core part of the `SELECT` statement and its variants, such as `SELECT INTO` and queries that use `WHERE`.

## Key Features of FROM

- Identifies the **source table(s)** for a query  
- Works in conjunction with the `SELECT` clause  
- Supports **joining multiple tables** to retrieve related data  
- Allows data to be pulled from **one or more tables**

## Using Multiple Tables

When retrieving data from multiple tables:
- Table names are typically **separated by commas** (older syntax)
- Modern SQL commonly uses **JOIN clauses** for better readability and control

## Summary

The `FROM` clause defines where the data comes from and plays a crucial role in executing SQL queries that retrieve and combine data from one or more tables.
