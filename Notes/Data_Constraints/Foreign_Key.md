# Foreign Key

A **foreign key** in SQL is a column or a group of columns in one table that references the **primary key** of another table. It establishes a relationship between two tables and helps maintain consistency across related data.

## Key Characteristics

- References the **primary key** of another table  
- Can consist of **one or more columns**  
- Enforces **referential integrity** between tables  

## Purpose of a Foreign Key

Foreign keys help:
- Ensure values in the referencing table match valid values in the referenced table  
- Prevent **orphaned records**
- Maintain **data consistency** across related tables  

## Role in Database Design

Foreign keys are crucial for:
- Implementing **relational database designs**
- Defining relationships between tables  
- Supporting **complex queries** that involve joining multiple tables  

## Summary

Foreign keys play a vital role in relational databases by preserving data integrity and enabling meaningful relationships between tables.
