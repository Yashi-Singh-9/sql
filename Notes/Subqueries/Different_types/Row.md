# Row

In SQL, a **row** (also known as a **record** or **tuple**) represents a single, implicitly structured data item in a table. Each row contains a collection of related values, with each value corresponding to a column in the table.

## Key Characteristics

- Represents **one complete record** in a table  
- Contains data aligned with the table’s **column structure**  
- Each row holds related information about a single entity  

## Role in SQL Operations

Most data manipulation operations work at the row level, including:
- `INSERT` – adds new rows  
- `UPDATE` – modifies existing rows  
- `DELETE` – removes rows  

## Importance of Rows

Rows are fundamental to the **relational database model**, enabling:
- Organized storage of information  
- Efficient data retrieval  
- Structured data manipulation  

## Summary

Rows form the core of data storage in SQL tables, representing individual records that can be inserted, updated, queried, and deleted as part of database operations.
