# Primary Key

A **primary key** in SQL is a unique identifier for each record in a database table. It ensures that every row in the table can be uniquely identified.

## Key Characteristics

- Uniquely identifies each record in a table  
- Can consist of **one or more columns**  
- Must contain **unique values**  
- Cannot contain `NULL` values  

## Purpose of a Primary Key

The primary key enforces **entity integrity** by:
- Preventing **duplicate records**
- Ensuring each record can be **precisely located**
- Allowing reliable **references from other tables** through foreign keys

## Role in Database Design

Primary keys are fundamental for:
- Establishing **relationships between tables**
- Supporting **foreign key constraints**
- Maintaining the overall **integrity of the data model**

## Summary

Using a primary key is essential in relational databases to ensure data accuracy, consistency, and reliable relationships between tables.
