# What Are Relational Databases?

Relational databases are a type of **Database Management System (DBMS)** that store and provide access to data points that are related to one another. Based on the **relational model** introduced by **E. F. Codd in 1970**, they organize data into structured tables consisting of rows and columns.

## Key Features of Relational Databases

- Use of **SQL (Structured Query Language)** for querying and managing data  
- Support for **ACID transactions**:
  - **Atomicity**
  - **Consistency**
  - **Isolation**
  - **Durability**
- Enforcement of **data integrity** through constraints such as:
  - Primary keys
  - Foreign keys
- Ability to establish **relationships between tables**, enabling complex queries and efficient data retrieval
- **Scalability** and support for **multi-user environments**

## Examples of Relational Database Systems

Some popular relational database systems include:
- MySQL
- PostgreSQL
- Oracle
- Microsoft SQL Server

Relational databases are widely used across various applications, ranging from small-scale projects to large enterprise systems, due to their **reliability**, **consistency**, and **powerful querying capabilities**.
