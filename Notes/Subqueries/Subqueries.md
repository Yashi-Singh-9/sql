# Subqueries

Subqueries, also known as **nested queries** or **inner queries**, are SQL queries embedded within another SQL query. They enable complex data retrieval and manipulation by allowing queries to be broken down into smaller, more manageable parts.

## Where Subqueries Can Be Used

Subqueries can appear in various parts of an SQL statement, including:
- `SELECT`
- `FROM`
- `WHERE`
- `HAVING`

## Purpose of Subqueries

Subqueries are useful for:
- Creating **dynamic conditions** based on query results
- Performing **calculations** within a query
- **Comparing result sets** from different queries
- Simplifying complex SQL logic

## How Subqueries Work

The inner query is executed first, and its result is then used by the outer query to produce the final output.

## Summary

Subqueries are a powerful SQL feature that enhance flexibility and readability, making it easier to construct advanced queries and perform sophisticated data analysis.
